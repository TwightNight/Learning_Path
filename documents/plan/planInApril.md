# 🗓️ Weekly Git & GitHub Practice Plan

## 🎯 Goal
- Master basic Git commands
- Understand real workflow (commit, branch, PR)
- Build habit of clean commits

---

## 📅 Day 1 – Setup & First Repo
- [ ] Create GitHub account
- [ ] Create repo: `git-practice`
- [ ] Clone repo to local
- [ ] Create `README.md`
- [ ] Make first commit: `feat: initial commit`
- [ ] Push to GitHub

---

## 📅 Day 2 – Basic Commands
- [ ] Practice: `git add`, `git commit`, `git push`
- [ ] Create folder `day2-basic`
- [ ] Add 2–3 files
- [ ] Make at least 3 commits
- [ ] Use proper commit messages

---

## 📅 Day 3 – Pull & Update Code
- [ ] Practice: `git pull`
- [ ] Update existing files
- [ ] Add new file `notes.txt`
- [ ] Commit changes with clear messages
- [ ] Push updates

---

## 📅 Day 4 – Branching
- [ ] Create branch: `feature/day4`
- [ ] Add new files in branch
- [ ] Commit changes
- [ ] Push branch to GitHub
- [ ] Switch back to `main`

---

## 📅 Day 5 – Merge & Pull Request
- [ ] Create Pull Request on GitHub
- [ ] Review changes
- [ ] Merge branch into `main`
- [ ] Delete branch after merge
- [ ] Pull latest code to local

---

## 📅 Day 6 – Conflict Practice
- [ ] Create 2 branches editing same file
- [ ] Merge → cause conflict
- [ ] Resolve conflict manually
- [ ] Commit resolved version
- [ ] Push final code

---

## 📅 Day 7 – Clean Project & Review
- [ ] Organize folder structure
- [ ] Update `README.md`:
  - Project description
  - How to run
- [ ] Review commit history
- [ ] Ensure commit messages are clean
- [ ] Push final version

---

## ✅ Final Checklist
- [ ] 10+ commits
- [ ] 2+ branches used
- [ ] 1 Pull Request merged
- [ ] Conflict resolved at least once
- [ ] Clean README

---

## 🚀 Bonus (Optional)
- [ ] Learn `.gitignore`
- [ ] Try `git log`
- [ ] Try `git stash`
